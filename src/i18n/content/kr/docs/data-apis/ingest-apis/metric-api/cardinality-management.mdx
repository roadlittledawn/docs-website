뉴럴릭의 관리자로서 여러분의 업무 중 하나는 [수집 가능한 GB](/docs/licenses/license-information/product-definitions/new-relic-one-pricing-definitions/#gb-ingested) 한도를 초과하지 않도록 하는 것입니다. 수집을 추적하는 한 가지 도구는 Cardinality Management UI입니다. 섭취량을 설명하는 유용한 그래프와 표가 제공되므로 한계에 도달하면 조치를 취할 수 있습니다.

## 기수를 어디에서 관리할 수 있나요? [#where-manage]

뉴렐릭 UI 에서는 다른 관리 도구와 함께 이 기능을 찾을 수 있습니다. 왼쪽 하단에서 이름을 클릭하고 **Administration &gt; Cardinality Management** 로 이동합니다.

## 계정 기수 분석 [#account-breakdown]

여러 계정이 있는 조직에 속해 있다면 각 계정별 기수별 세부 정보를 볼 수 있습니다. 뉴렐릭의 대부분 기능과 마찬가지로, 오른쪽 상단 모서리에 있는 계정 선택기를 선택하여 현재 보고 있는 계정을 변경할 수 있습니다. 해당 계정에 액세스할 수 있는 한, UI에는 선택한 계정에 대한 정보가 표시됩니다.

### 시간 경과에 따른 사용 [#usage-over-time]

**Cardinality Management** 의 시작 페이지에서 **Usage over time** \[시간 경과에 따른 사용량] 차트를 볼 수 있습니다.

<img style={{ align: 'left',maxWidth: '90%' }} title="Screenshot showing the usage over time chart" alt="Screenshot showing the usage over time chart" src="/images/apis-and-data_screenshot-crop_usage-over-time.webp" />

이는 다음을 보여줍니다:

* 현재 한도는 계정 카디널리티 500만 개입니다.
* 해당 계정의 현재 카디널리티 사용량은 200만-400만 범위입니다. 값에 변화가 없으면 새로운 데이터 포인트는 보고되지 않습니다. 이는 빈 데이터 포인트로 나타날 수 있지만 데이터 손실은 아닙니다. 이러한 동작은 기능이 완성됨에 따라 변경될 수 있습니다.

<Callout variant="tip">
  오른쪽 상단 모서리에 있는 타임 피커를 변경하여 이 시계열을 수정할 수 있습니다. 또한, 클럭 피커는 시계열에만 영향을 미치며 시계열 아래의 **Metrics** \[메트릭] 표에는 영향을 미치지 않습니다.
</Callout>

### 메트릭 테이블 [#metrics-table]

**Metrics** \[지표] 테이블에서 지표 이름별 계정 기수 분류를 확인할 수 있습니다.

<img style={{ align: 'left',maxWidth: '90%' }} title="Screenshot showing the metrics table" alt="Screenshot showing the metrics table" src="/images/apis-and-data_screenshot-crop_metrics-table.webp" />

이 표는 계정 내 상위 20개의 높은 카디널리티 지수를 보여줍니다. 표에는 다음과 같은 세부 정보가 나와 있습니다.

* 메트릭의 이름
* 현재 메트릭의 전체 사용량(현재 사용량과 현재 한도의 비율)
* 현재 기수의 실제 값
* 해당 메트릭 이름의 현재 제한

<Callout variant="tip">
  모든 지표의 기본 지표 한도는 100,000입니다.
</Callout>

## 메트릭 카디널리티 분석 [#metric-cardinality-breakdown]

지표 이름을 클릭하면 지표에서 더욱 자세한 세부 정보를 볼 수 있습니다. 특정 지표의 기수 세부 정보에 대한 분석을 볼 수 있습니다.

<img style={{ align: 'left',maxWidth: '90%' }} title="Screenshot showing the Cardinality Management page" alt="Screenshot showing the Cardinality Management page" src="/images/apis-and-data_screenshot-crop_metric-cardinality-breakdown.webp" />

### 지표 및 계정 카디널리티 바 차트 [#bar-charts]

이 측정항목별 보기에서는 현재 카디널리티 사용량을 **Metric cardinality budget** \[측정항목 카디널리티 예산]과 **Account cardinality budget** \[계정 카디널리티 예산]\(전체 계정 사용량)에 대한 두 개의 막대 차트로 볼 수 있습니다.

## 시간 경과에 따른 사용(표적용) [#usage-over-time-metrics]

막대 차트 아래에는 지수 카디널리티 사용량과 해당 지수의 한계를 보여주는 시계열 쿼리 차트가 있습니다. 다시 말해, 타임피커를 변경하여 개별 지표의 기수 데이터의 다른 기간을 볼 수 있습니다.

<Callout variant="tip">
  기수 한도(일반적으로 100,000)를 초과한 모든 지표의 경우 뉴렐릭은 여전히 지표에 정보를 채우려고 시도하지만, 집계된 데이터 대신 RAW 데이터를 사용하므로 약간 더 느립니다.
</Callout>

### 속성 테이블 [#attributes-table]

시계열 아래에 메트릭 카디널리티를 구동하는 속성을 보여주는 표가 있습니다. 기수는 여러 속성의 조합으로 계산되기 때문에 숫자가 일반적으로 100%에 도달하지 않는다는 점에 유의하세요. 하지만 이는 귀하의 가장 큰 기여 속성이 무엇인지를 나타내는 좋은 지표입니다.

<img style={{ align: 'left',maxWidth: '90%' }} title="Screenshot showing attributes behind metrics" alt="Screenshot showing attributes behind metrics" src="/images/apis-and-data_screenshot-crop_metric-attributes.webp" />

이 정보는 마지막 날(UTC 자정 이후)에만 적용되며, 이때 기수(계정 및 지표 모두)가 재설정됩니다.

또한 속성에 대한 구체적인 정보를 볼 수 있으며, 이는 해당 속성이 유용한지 여부를 결정하는 데 도움이 될 수 있습니다. 속성 행의 오른쪽에 있는 메뉴를 클릭하면 속성 값의 샘플을 볼 수 있습니다.

## 계정 정리 규칙 [#account-pruning-rules]

계정 기수량을 한도 내로 유지하는 일반적인 방법은 중요하지 않다고 생각되는 메트릭과 속성에 대한 정리 규칙을 구현하는 것입니다.

### 계정 정리 규칙 보기 [#view-pruning-rules]

측정항목을 클릭한 다음 **View Pruning Rules** \[정리 규칙 보기]를 클릭하면 정리 규칙을 볼 수 있습니다.이렇게 하면 계정과 관련된 모든 정리 규칙이 스크롤 가능한 목록으로 표시됩니다. 이 보기에서 계정과 관련된 정리 규칙을 삭제할 수도 있습니다.

<img style={{ align: 'left',maxWidth: '90%' }} title="Screenshot showing the link to pruning rules on the Attributes page" alt="Screenshot showing the link to pruning rules on the Attributes page" src="/images/apis-and-data_screenshot-crop_view-pruning-rules.webp" />

### 가지치기 규칙 만들기 [#create-pruning-rule]

UI 내의 속성을 기반으로 가지치기 규칙을 만들 수 있습니다. 가지치기 규칙을 만들려면:

1. 정리하려는 속성 옆에 있는 확인란을 선택하세요.

   <img style={{ align: 'left', maxWidth: '90%' }} title="Screenshot showing how to select attributes for pruning rule" alt="Screenshot showing how to select attributes for pruning rule" src="/images/apis-and-data_screenshot-crop_select-attributes-for-pruning-rule.webp" />

2. **Create pruning rule** \[가지치기 규칙 만들기를] 클릭합니다.

3. 가지치기 규칙이 지표의 기수에 어떻게 영향을 미치는지에 대한 정보가 있는 팝업에서 규칙 이름이나 설명을 입력합니다.

   <img style={{ align: 'left' }} title="Screenshot of page to name the pruning rule" alt="Screenshot of page to name the pruning rule" src="/images/apis-and-data_screenshot-crop_name-pruning-rule.webp" />

4. **Submit** \[제출을] 클릭하여 가지치기 규칙을 생성합니다.

가지치기 규칙은 즉각적으로 적용된다는 점을 알아두는 것이 중요합니다. 규칙을 만든 직후에 정보가 즉시 정리되므로 데이터를 정리할 것인지 반드시 확인하세요.

가지치기 규칙을 만든 경우 규칙을 만든 후에도 규칙을 삭제할 수 있지만, 삭제하기 전에 규칙에 의해 약간의 데이터가 제거될 수 있습니다.

### 가지치기 규칙 삭제 [#delete-pruning-rule]

가지치기 규칙을 삭제하려면:

1. **View All Pruning Rules** \[모든 가지치기 규칙 보기를] 클릭하세요.
2. 해당 가지치기 규칙으로 스크롤한 다음 **Delete** \[삭제를] 클릭합니다.
